# 中文 Paper Note 模板

```markdown
# Paper Note

## 输入覆盖范围
- 来源：
- 覆盖级别：fulltext / partial-text / abstract / metadata / user-notes
- 边界说明：

## 论文元信息
- 标题：
- 作者：
- 年份 / Venue：
- 链接：
- 主题标签：

## 一句话
用一句大白话说清楚这篇论文在干嘛。

## 直觉版讲解：像给刚入门同学讲
- 这篇论文想解决的麻烦：
- 它的核心想法像什么：
- 它具体怎么做，分几步：
  1.
  2.
  3.
- 类比和真实技术的对应关系：
  - 类比里的 A 对应论文里的：
  - 类比里的 B 对应论文里的：
- 这个想法为什么可能有效：

## 论文结构：每个 section 大概在说什么
- Abstract：
- Introduction：
- Related Work / Background：
- Method：
- Experiments：
- Analysis / Ablation：
- Conclusion / Limitation：

## 核心 Insight
- 最有价值的 insight：
- 它和已有工作的关键差别：
- 这个 insight 能不能迁移到我们的项目：

## 方法简图
- 输入：
- 中间过程：
- 输出：
- 关键模块或机制：

## 实验在证明什么
- 主要实验想证明：
- 最关键的结果：
- 消融或分析说明：
- 还有什么没证明清楚：

## 对当前项目的启发
- 可以直接借鉴：
- 可以作为 baseline / 对照：
- 可以写进 related work 的角度：
- 可以启发的新实验或新 idea：
- 可能作为反例或边界案例：

## 可引用点 / 可比较点
- 一句话引用方式：
- 应该和哪些论文放在一起比较：
- 适合放进 taxonomy 的哪一类：

## 疑问和局限
- 我还没搞清楚：
- 论文可能没覆盖：
- 后续需要补读或验证：
```

## 使用说明

- 默认保持短，不要为了填满模板写废话。
- 如果某个 section 在输入里没有信息，写 `未知` 或 `输入未覆盖`。
- “直觉版讲解”要生动，但必须能映射回真实技术组件。
- “对当前项目的启发”要写成具体动作或判断，不要只写“有启发”。
- 如果只是摘要级输入，实验和局限部分要明显保守。
